# promptname_a

System-prompt experiment built from the authorized agentic-loop and prompt-architecture writeups.

Intent:

- keep OPPi's current user-facing tone;
- make the agent's internal operating loop more explicit;
- improve markdown structure and diagram use without making responses noisy;
- preserve Pi/OPPi runtime boundaries instead of pretending OPPi owns the whole loop.

Runtime entrypoint: `main-system-append.md`, selectable with `/prompt-variant promptname_a`.
